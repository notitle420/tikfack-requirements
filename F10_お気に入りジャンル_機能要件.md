# F10 お気に入り（ジャンル） 機能要件

## 1. 目的
- ログインユーザーの好みのジャンルを明示的に保存し、マイページで閲覧できるようにする。
- ジャンル情報をレコメンドに反映する。

## 2. 対象範囲・前提
- 年齢確認（F01）通過済みかつログイン済み（F05）であること。
- 対象はジャンル単位とする。

## 3. 機能概要
- ジャンル一覧や動画のジャンル表示から、お気に入りジャンルの登録／解除を行う。
- マイページでお気に入りジャンル一覧を閲覧できる。

## 4. 詳細要件
- アクセス制御
  - お気に入りジャンルの登録／解除および一覧取得はログイン必須。
- 登録／解除
  - 同一ジャンルの重複登録を防止する。
  - 操作後は UI に即時反映し、バックエンドと状態を同期する。
- 一覧表示
  - マイページでお気に入りジャンルの一覧を表示し、ジャンルから動画一覧（検索など）へ遷移できるようにする。
  - 並び順は追加日時降順。ソート/フィルタは提供しない。
- 上限
  - 登録上限は設けない。
- ロギング
  - 追加／解除操作をユーザー ID、ジャンル ID とともにログ化し、レコメンド（F09）に利用可能とする。

## 5. 入出力
- 入力: ジャンル ID、ユーザー ID、登録／解除リクエスト。
- 出力: 処理結果ステータス、更新後のお気に入り状態、一覧取得時はジャンルリスト。

## 6. 関連
- F07 検索: ジャンル選択からの動画検索。
- F04 マイページ: お気に入りジャンルの一覧表示。
- F09 レコメンド: ジャンル嗜好として利用。
