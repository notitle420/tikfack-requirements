# F10 レコメンド 機能要件

## 1. 目的
- 視聴履歴やお気に入り情報を利用して、ログインユーザーに適した動画を優先表示し、視聴体験と外部サービスへの誘導効率を高める。

## 2. 対象範囲・前提
- 年齢確認（F01）通過済みかつログイン済み（F05）であること。
- レコメンド対象はトップ画面の動画リストとする。

## 3. 機能概要
- ログインユーザーのトップ画面に対し、視聴ログ（F09）やお気に入り（F06/F12/F13）情報を元に算出した動画リストを返却する。
- レコメンドが形成できない場合は、最新／人気といったデフォルトリストを返却する。

## 4. 詳細要件
- 入力データ
  - 視聴ログ（視聴回数、完了率、直近視聴など）。
  - お気に入り動画／ジャンル／女優情報。
- 出力
  - レコメンドスコアに基づき並び替えられた動画リスト。
  - ページネーション情報（トップ画面の縦スクロール用）。
- ロジック
  - アルゴリズムの詳細（重みづけ、期間など）は設計フェーズで定義する。
  - 当初は単純なルールベース（最近よく見ているジャンル・女優を優先）から開始することを想定。

## 5. 入出力
- 入力: ユーザー ID、視聴ログ／お気に入り情報（内部的に参照）。
- 出力: レコメンド動画リスト（メタ情報、サンプル動画 URL、ページネーション情報）。

## 6. 関連
- F02 縦スクロール視聴・レコメンド: トップ画面でレコメンドリストを表示。
- F06/F12/F13 お気に入り: レコメンドの入力情報。
- F09 視聴ログ: レコメンドの入力情報。
