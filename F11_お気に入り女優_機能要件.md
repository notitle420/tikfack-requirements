# F11 お気に入り（女優） 機能要件

## 1. 目的
- ログインユーザーの好みの女優を保存し、マイページで閲覧できるようにする。
- 女優情報をレコメンドに反映する。

## 2. 対象範囲・前提
- 年齢確認（F01）通過済みかつログイン済み（F05）であること。
- 対象は女優単位とする。

## 3. 機能概要
- 動画カードの出演者表示や女優詳細から、お気に入り女優の登録／解除を行う。
- マイページでお気に入り女優一覧を閲覧できる。

## 4. 詳細要件
- アクセス制御
  - お気に入り女優の登録／解除および一覧取得はログイン必須。
- 登録／解除
  - 同一女優の重複登録を防止する。
  - 操作後は UI に即時反映し、バックエンドと状態を同期する。
- 一覧表示
  - マイページでお気に入り女優の一覧を表示し、女優別の動画一覧（検索など）へ遷移できるようにする。
- ロギング
  - 追加／解除操作をユーザー ID、女優 ID とともにログ化し、レコメンド（F09）に利用可能とする。

## 5. 入出力
- 入力: 女優 ID、ユーザー ID、登録／解除リクエスト。
- 出力: 処理結果ステータス、更新後のお気に入り状態、一覧取得時は女優リスト。

## 6. 関連
- F07 検索: 女優名からの検索。
- F04 マイページ: お気に入り女優の一覧表示。
- F09 レコメンド: 女優嗜好として利用。
