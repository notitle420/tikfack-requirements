# F06 お気に入り（動画） 機能要件

## 1. 目的
- ログインユーザーが気に入った動画を保存し、後から簡単に閲覧できるようにする。
- お気に入り情報をレコメンドや分析に活用する。

## 2. 対象範囲・前提
- 年齢確認（F01）通過およびログイン済み（F05）であること。
- 対象は各動画単位とする。

## 3. 機能概要
- 動画カード上からお気に入り登録／解除を行える。
- マイページの「お気に入り動画」一覧で登録済み動画を閲覧できる。

## 4. 詳細要件
- アクセス制御
  - お気に入り登録／解除、お気に入り一覧取得はログイン必須。
- 登録／解除
  - 動画カード上のアイコンやボタンからお気に入り登録／解除を行う。
  - 操作後は UI 上の状態を即時に切り替え、バックエンドと非同期で整合性を取る。
- 一覧表示
  - マイページに「お気に入り動画」セクションを設け、追加日時降順で表示する。
  - 一覧から直接動画視聴画面（F02）へ遷移できる。
- ロギング
  - 追加／解除操作をユーザー ID、動画 ID とともにログとして保存し、レコメンド（F10）に利用可能とする。

## 5. 入出力
- 入力: 動画 ID、ユーザー ID（JWT から取得）、登録／解除リクエスト。
- 出力: 処理結果ステータス、更新後のお気に入り状態、一覧取得時は動画リスト。

## 6. 関連
- F02 縦スクロール視聴・レコメンド: 視聴画面からお気に入り操作を行う。
- F04 外部サービス遷移: お気に入り動画から外部への誘導も想定。
- F11 マイページ: お気に入り動画一覧を表示。
- F10 レコメンド: お気に入り情報をスコアリングに利用。
